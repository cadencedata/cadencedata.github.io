Cadence

Handle and access dynamic, distributed data across devices with ease.

Data is hard.

Working with a single dataset is non-trivial. You need to provision compute nodes, set up a database, and perform a data pull to write to that database — and that's before you even start thinking about opening a database connection, writing SQL, and pulling data into your preferred analysis environment.

Dynamic, distributed data is harder.

The complexity scales exponentially with multiple dynamic datasets, each of which must be periodically updated to reflect some data source. To handle this case, you need to manually or programmatically trigger data pulls to the appropriate database. Your use case might even involve joins or merges across multiple databases.

All your data operations should be in one place.

Cadence is a hub for all your dynamic, distributed data operations. Write lambdas — operations that write to a database table — in Python, then schedule jobs to execute your lambdas at specified intervals and perform those writes. Monitor the health and activity of all of your jobs, compute nodes, and databases through Cadence's interface.

You should have a hassle-free API into your own data.

With Cadence, there's no need to include messy, meta-programmed SQL in your analysis scripts. Cadence lets you define SQL hooks for a specific job's data results, allowing you to write your query once and abstract it away. You can pull your data into your desired analysis environment with the hook you defined simply by hitting an API endpoint. Cadence even allows you to parametrize hooks for greater flexibility.
